## Status: Done

## Story

**As a** user of claude-auto-resume,  
**I want** the tool to detect and handle various error conditions gracefully,  
**so that** I receive helpful error messages and the tool doesn't fail silently or get stuck.

3. Handle network timeout scenarios gracefully
4. Detect and handle malformed Claude CLI output

6. Provide recovery suggestions for common error scenarios
7. Ensure proper cleanup on error conditions

### Implementation Details

**Network Connectivity Check**: [Source: docs/ROADMAP.md#enhanced-error-handling]
```bash
# Basic connectivity check before Claude operations
if ! ping -c 1 8.8.8.8 >/dev/null 2>&1; then
    echo "[WARNING] Network connectivity check failed. Claude CLI may not work properly."
    echo "Please check your internet connection and try again."
fi
```

**Enhanced Error Messages**: [Source: docs/architecture.md#architectural-and-design-patterns]
- Provide clear, actionable error messages
- Include debugging hints for common failure scenarios
- Use existing error handling patterns from the script
- Add context information to help users troubleshoot

**Timeout Protection**: [Source: docs/ROADMAP.md#enhanced-error-handling]
```bash
# Add timeout for Claude CLI operations
timeout 30s claude -p 'check' 2>&1 || {
    echo "[ERROR] Claude CLI operation timed out after 30 seconds"
    echo "This may indicate network issues or Claude service problems"
    exit 3
}
```

**Error Code Strategy**: [Source: docs/architecture.md#architectural-and-design-patterns]
- Exit code 1: Claude CLI execution failures (existing)
- Exit code 2: Invalid timestamp extraction (existing)
- Exit code 3: Network/timeout issues (new)
- Exit code 4: Resume command failures (existing)
- Exit code 5: Malformed output/parsing errors (new)

- Maintain dependency on `claude` CLI but add validation for its responses
